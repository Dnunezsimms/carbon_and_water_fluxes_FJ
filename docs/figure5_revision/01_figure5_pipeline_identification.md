## Traceability notes

- `docs/workflow.md` identifies Figure 5 as being driven by `scripts/figures/Figure_5.py` and `data/processed/LEVEL3_FJ_2022_2025_ss_v4_daily.csv`.
- The dataset header contains duplicated `TIMESTAMP` column names. For this review, timestamps were read from the first column explicitly to avoid ambiguous CSV import behavior.
- The current repository script already contains `PAR = PAR.where(PAR > 0)`, which means the exported repository figure is consistent with leaving PAR gaps blank.
- There is a manuscript-to-repository discrepancy worth noting: the manuscript description says panel a includes Ta and PAR, but the current repository script and current exported figure show PAR only in panel a.
